## 📝 推荐启动方式（最简单）

### 使用 IntelliJ IDEA 运行（强烈推荐）

#### 步骤 1：打开项目
1. 打开 IntelliJ IDEA
2. 点击 `File` → `Open`
3. 选择 `d:\uisheji\javaproject` 目录
4. 点击 `Open`

#### 步骤 2：等待依赖下载
- IDEA 会自动识别 `pom.xml` 并开始下载 Maven 依赖
- 首次运行需要 **2-5 分钟**（取决于网络速度）
- 右下角会显示进度条

#### 步骤 3：运行项目
1. 在左侧项目树中找到：
   ```
   src/main/java/com/uisheji/UishejiBackendApplication.java
   ```
2. 右键点击该文件
3. 选择 `Run 'UishejiBackendApplication'`

#### 步骤 4：访问应用
- **服务地址**: http://localhost:8080/api
- **API 文档**: http://localhost:8080/api/swagger-ui/index.html
- **健康检查**: http://localhost:8080/api/actuator/health

## 🐛 常见问题

### Q1: IDEA 下载依赖很慢怎么办？
**A**: 配置国内 Maven 镜像源
1. 打开 `Settings` → `Build, Execution, Deployment` → `Build Tools` → `Maven`
2. 点击 `User settings file` 的 `Edit`
3. 添加阿里云镜像：
```xml
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>*</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### Q2: 端口 8080 被占用？
**A**: 修改 `src/main/resources/application.yml` 中的端口配置：
```yaml
server:
  port: 8081  # 改成其他端口
```

### Q3: 数据库连接失败？
**A**: 检查数据库配置是否正确：

- 端口：3306

## 📚 项目结构

```
javaproject/
├── src/main/java/com/uisheji/
│   ├── config/           # 配置类（安全、Swagger）
│   ├── controller/       # 控制器（AuthController、UserController）
│   ├── dto/             # 数据传输对象（请求、响应）
│   ├── entity/          # 实体类（User）
│   ├── exception/       # 异常处理（BusinessException）
│   ├── repository/      # 数据访问层（UserRepository）
│   ├── security/        # 安全相关（JWT Filter）
│   ├── service/         # 业务逻辑层
│   └── util/           # 工具类（JwtUtil）
└── src/main/resources/
    ├── application.yml  # 配置文件
    └── db/migration/   # 数据库迁移脚本
```
