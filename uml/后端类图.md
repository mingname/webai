# 后端类图 (Backend Class Diagram)

## 1. 概述

后端采用 MVC（Model-View-Controller）架构模式，使用 Express 框架构建 RESTful API。

**技术栈：**
- Node.js
- Express 4.18
- MySQL 5.7+
- JWT 认证
- bcryptjs 密码加密

## 3. 类图详解

### 3.1 服务器应用类

```mermaid
classDiagram
    class ExpressApp {
        -Express app
        -Number port
        +use(middleware)
        +get(path, handler)
        +post(path, handler)
        +listen(port, callback)
    }
    
    class Server {
        +init()
        +start()
        +errorHandler()
    }
    
    ExpressApp --> Server
```

**属性说明：**
- `app`: Express 应用实例
- `port`: 服务器端口（默认 3001）

**方法说明：**
- `init()`: 初始化服务器配置和中间件
- `start()`: 启动服务器监听
- `errorHandler()`: 全局错误处理

**方法说明：**

### 3.3 认证中间件类

```mermaid
classDiagram
    class AuthMiddleware {
        +String secret
        +String expiresIn
        +authenticateToken(req, res, next): void
        +generateToken(user): String
    }
    
    class JWT {
        +sign(payload, secret, options): String
        +verify(token, secret, callback): void
    }
    
    class TokenPayload {
        +Number id
        +String username
        +String nickname
    }
    
    AuthMiddleware ..> JWT : 使用
    AuthMiddleware ..> TokenPayload : 创建
    
    TokenPayload --|> Object
```

**属性说明：**
- `secret`: JWT密钥（从环境变量读取）
- `expiresIn`: Token过期时间（默认 24h）

**方法说明：**

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `authenticateToken` | req, res, next | void | 验证JWT token，将用户信息附加到req.user |
| `generateToken` | user对象 | String token | 生成JWT token |

**Token结构：**
```javascript
{
  id: 用户ID,
  username: 用户名,
  nickname: 昵称,
  iat: 签发时间,
  exp: 过期时间
}
```

## 4. 时序图

### 4.1 用户登录流程

```mermaid
sequenceDiagram
    participant C as 客户端
    participant R as 路由层
    participant V as 验证中间件
    participant A as 认证中间件
    participant Ctrl as 控制器
    participant DB as 数据库
    participant JWT as JWT服务
    participant Bcrypt as Bcrypt服务
    
    C->>R: POST /api/login {username, password}
    R->>V: 验证参数
    V-->>R: 验证通过
    R->>Ctrl: login(req, res)
    Ctrl->>DB: SELECT * FROM users WHERE username = ?
    DB-->>Ctrl: 用户记录
    Ctrl->>Bcrypt: compare(password, hashedPassword)
    Bcrypt-->>Ctrl: true/false
    alt 密码正确
        Ctrl->>DB: UPDATE users SET last_login_time = NOW()
        Ctrl->>JWT: sign({id, username, nickname})
        JWT-->>Ctrl: token
        Ctrl-->>C: {code: 200, token, userInfo}
    else 密码错误
        Ctrl-->>C: {code: 401, message: "用户名或密码错误"}
    end
```

### 4.2 用户注册流程

```mermaid
sequenceDiagram
    participant C as 客户端
    participant R as 路由层
    participant Ctrl as 控制器
    participant DB as 数据库
    participant Bcrypt as Bcrypt服务
    
    C->>R: POST /api/register {username, password, nickname}
    R->>Ctrl: register(req, res)
    Ctrl->>DB: SELECT id FROM users WHERE username = ?
    DB-->>Ctrl: 空结果
    Ctrl->>Bcrypt: hash(password, 10)
    Bcrypt-->>Ctrl: hashedPassword
    Ctrl->>DB: INSERT INTO users VALUES (...)
    DB-->>Ctrl: insertId
    Ctrl-->>C: {code: 201, id, username, nickname, avatar}
```

### 4.3 获取用户信息流程

```mermaid
sequenceDiagram
    participant C as 客户端
    participant R as 路由层
    participant A as 认证中间件
    participant JWT as JWT服务
    participant Ctrl as 控制器
    participant DB as 数据库
    
    C->>R: GET /api/user/info<br/>Authorization: Bearer {token}
    R->>A: authenticateToken(req, res, next)
    A->>JWT: verify(token, secret)
    JWT-->>A: payload {id, username, nickname}
    A-->>R: req.user = payload
    R->>Ctrl: getUserInfo(req, res)
    Ctrl->>DB: SELECT * FROM users WHERE id = ?
    DB-->>Ctrl: 用户记录（不含密码）
    Ctrl-->>C: {code: 200, userInfo}
```

## 7. 错误处理流程

```mermaid
graph TB
    Start[请求开始] --> Middleware[中间件处理]
    Middleware --> Validator[参数验证]
    
    Validator -->|验证失败| ValidationError[返回400错误]
    Validator -->|验证通过| Auth[认证检查]
    
    Auth -->|Token无效| AuthError[返回401/403错误]
    Auth -->|Token有效| Controller[控制器处理]
    
    Controller --> Database[数据库操作]
    Database -->|查询失败| DBError[返回500错误]
    Database -->|查询成功| Success[处理成功]
    
    Controller -->|业务逻辑错误| BusinessError[返回业务错误]
    Controller -->|成功| Success
    
    Success --> Response[返回成功响应]
    ValidationError --> Response
    AuthError --> Response
    DBError --> Response
    BusinessError --> Response
```
